# H5页面显示问题排查

## 问题：页面显示目录列表而不是应用内容

如果你看到的是目录列表页面（显示文件列表），说明开发服务器没有正确启动或访问的URL不正确。

## 解决方案

### 1. 确保开发服务器正在运行

在终端中应该看到类似这样的输出：
```
👽 Taro v3.6.38
✓ 编译成功
✓ 监听文件修改中...
```

### 2. 访问正确的URL

**正确的URL：**
```
http://localhost:10086
```

**不要访问：**
- `http://localhost:10086/` (根目录)
- `file:///` 开头的本地文件路径

### 3. 如果仍然显示目录列表

#### 步骤1：停止当前服务器
按 `Ctrl+C` 停止开发服务器

#### 步骤2：清理构建文件
```bash
cd frontend
rm -rf dist .temp
```

#### 步骤3：重新启动
```bash
npm run dev:h5
```

#### 步骤4：等待编译完成
等待看到 "✓ 编译成功" 的消息

#### 步骤5：访问URL
在浏览器中访问：`http://localhost:10086`

### 4. 检查浏览器控制台

打开浏览器开发者工具（F12），检查：
- **Console标签**：是否有错误信息
- **Network标签**：资源是否成功加载
- **Elements标签**：是否有页面内容

### 5. 常见问题

#### 问题1：端口被占用
如果10086端口被占用，Taro会提示使用其他端口，注意终端输出的实际端口号。

#### 问题2：编译失败
检查终端是否有编译错误，修复错误后重新启动。

#### 问题3：缓存问题
清除浏览器缓存或使用无痕模式访问。

### 6. 验证服务器是否正常运行

在终端中应该看到：
```
✓ 编译成功
✓ 监听文件修改中...

App running at:
- Local:   http://localhost:10086/
- Network: http://192.168.x.x:10086/
```

如果看到这些信息，说明服务器正常运行，访问显示的URL即可。
